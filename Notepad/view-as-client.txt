BEBERAPA KONDISI
1. biasa
kondisi: root view as agency | root view as client | agency view as client (new)
localStorage: 
    1. userData
    2. userDataOri

2. special
- kondisi pertama: root view as agency
  localStorage: 
    1. userData
    2. userDataOri
- kondisi kedua: lalus setelah itu view as client
  localStorage: 
    1. userData
    2. userDataOri
    3. userDataAgency

RULES
1. untuk switchUserData kan posisinya selalu dari userData. cara nya seperti ini
- pertama check dulu di localStorage ada userDataOri tidak. jika ada maka simpan di userDataAgency (ini terjadi ketika root view as agency, lalu view as lagi sebagai client) ✅
    * backup companyGroupSelected
    * backup subdomainAgency
    * backup rootcomp
- kedua jika tidak ada userDataOri. maka simpan di userDataOri (ini terjadi ketika login sebagai agency nya langsung dan view as sebagai client, root view as agency, root view as client, kaya biasa sekarang ini)

2. untuk revertViewMode kan posisinya selalu dari userData. cara nya seperti ini
- pertama check dulu di localStorage ada userDataAgency tidak. jika ada pakai itu dan selesai (ini terjadi ketika root view as agency, lalu view as lagi sebagai client) ✅
- kedua jika tidak ada userDataAgency. maka pakai userDataOri (ini terjadi ketika login sebagai agency nya langsung dan view as sebagai client)
